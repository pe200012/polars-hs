# Expression DSL Core Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Implement Phase 1 of upstream Polars expression parity: casts, predicates, fills, conditionals, statistics, cumulative expressions, ranking, expression slicing/filtering/sorting, and window `over`.

**Architecture:** Keep `Polars.Expr` as a pure Haskell AST. Compile each node into a short-lived Rust `Expr` handle at lazy-plan FFI boundaries. Add compact `phs_expr_*` Rust ABI families with private opcode mapping inside Haskell internal modules.

**Tech Stack:** Haskell 2010, Stack/Hpack, GHC 9.12.2, Rust 2024, Polars 0.53.0, cbindgen, Hspec, cargo test/clippy, hlint.

0 => expr.is_null()
1 => expr.is_not_null()
2 => expr.is_nan()
3 => expr.is_not_nan()
4 => expr.is_finite()
5 => expr.is_infinite()

13 => expr.cum_prod(true)

17 => expr.cum_max(true)

- [ ] **Step 3: Compile new AST nodes**

Extend `compileExpr` cases:

```haskell
Cast strict dtype expr -> compile child, encode dtype, call phs_expr_cast
UnaryExpr fn expr -> compile child, call phs_expr_unary or phs_expr_unary_i64
BinaryFunctionExpr FillNull input fill -> compile both, call phs_expr_binary_function 0
BinaryFunctionExpr FillNan input fill -> compile both, call phs_expr_binary_function 1
BinaryFunctionExpr ExprFilter input predicate -> compile both, call phs_expr_binary_function 2

QuantileExpr method quantile expr -> compile quantile and expr, call phs_expr_quantile
RankExpr options expr -> compile expr, call phs_expr_rank
SortByExpr options by expr -> compile expr and by list, call phs_expr_sort_by
OverExpr partition expr -> compile expr and partition list, call phs_expr_over
```
